# Разработка локально (проект перенесён с SSH-сервера)

Раньше приложение работало на сервере: в `.env` указан `DB_HOST=localhost` — это был MySQL на том же сервере. Сейчас проект у вас на ПК, и «localhost» указывает на ваш компьютер, где MySQL не запущен.

## Как сделать так, чтобы БД работала локально

Нужно поднять **SSH-туннель** с вашего ПК на сервер. Тогда подключение к `127.0.0.1:3306` на вашем компе будет перенаправляться на MySQL на сервере.

### Шаг 1. Запустить туннель

В отдельном окне PowerShell или терминале выполните (подставьте свой логин и хост сервера):

```powershell
ssh -L 3306:localhost:3306 ВАШ_ЛОГИН@СЕРВЕР
```

Примеры:
- `ssh -L 3306:localhost:3306 user@alumni.kazgasa.kz`
- или, если SSH на другом хосте: `ssh -L 3306:localhost:3306 user@server.kazgasa.kz`

Окно с SSH должно оставаться открытым, пока вы работаете с проектом.

**Если MySQL на сервере слушает не 3306**, а например 3307:

```powershell
ssh -L 3306:localhost:3307 ВАШ_ЛОГИН@СЕРВЕР
```

### Шаг 2. .env менять не нужно

В `.env` уже указано:
- `DB_HOST=localhost` (или `127.0.0.1`) и `DB_PORT=3306` — основная БД
- `IPORTAL_DB_HOST=127.0.0.1` и `IPORTAL_DB_PORT=3306` — БД iPortal

Пока туннель запущен, оба подключения будут идти через туннель на сервер. Если нужны **две** разные БД на сервере по разным портам, можно поднять два туннеля, например:

```powershell
# Окно 1: основная БД (alumni) — локально 3306 → сервер 3306
ssh -L 3306:localhost:3306 user@СЕРВЕР

# Окно 2: iPortal (moodle) — локально 3307 → сервер 3306
ssh -L 3307:localhost:3306 user@СЕРВЕР
```

Тогда в `.env` для iPortal: `IPORTAL_DB_PORT=3307` (и хост `127.0.0.1`).

### Шаг 3. Проверка

После запуска туннеля в папке проекта:

```powershell
php artisan db:show
php artisan db:show --database=iportal
```

Обе команды должны вывести данные о БД без ошибки подключения.

## Удобный запуск туннеля

В корне проекта есть скрипт `tunnel-db.ps1`. Откройте его, укажите в начале свои `SSH_USER` и `SSH_HOST`, сохраните и запускайте.

**Варианты:**

| Команда | Что делает |
|--------|------------|
| `.\tunnel-db.ps1` | Обычный запуск. Окно можно свернуть, но не закрывать. |
| `.\tunnel-db.ps1 -Reconnect` | При обрыве соединения туннель сам переподключается каждые 5 сек — не нужно вручную запускать заново. |
| `.\tunnel-db.ps1 -Background` | Запускает туннель в **отдельном свёрнутом окне**. Можно спокойно работать в другой консоли; не закрывайте то свёрнутое окно. |
| `.\tunnel-db.ps1 -Reconnect -Background` | Туннель в свёрнутом окне + автопереподключение при обрыве. |

Скрипт уже настроен на **keepalive** (раз в 60 сек пинг), поэтому соединение реже обрывается по таймауту.

## Для локальной разработки в браузере

Чтобы ссылки в приложении вели на ваш локальный сервер:

- В `.env` можно временно поставить:
  - `APP_ENV=local`
  - `APP_DEBUG=true`
  - `APP_URL=http://127.0.0.1:8766` (или порт, на котором у вас запущен `php -S ...`)

Не коммитьте `.env` с реальными паролями в репозиторий.
